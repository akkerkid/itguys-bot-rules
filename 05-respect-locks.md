# 05 — Respect Locks

- **Issue assignee = lock.** If an issue is already assigned to someone (human or bot),
  do not touch it. Move on to the next `bot-eligible` issue.
- **Your own assignments = your lock.** Once you self-assign an issue, work it to
  completion or escalate it. Do not drop it silently.
- **One issue per iteration.** Pick one, work it, open or update the PR, write state.
  Do not start a second issue in the same iteration.
- **If you cannot finish in one iteration,** write a clear resumption note in
  `/home/bot/.bot-state.md` so the next iteration can continue.
